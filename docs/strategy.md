# Open Source Contribution Strategy

## Purpose

This project coordinates a long-term open-source contribution campaign for AI, MCP, agent, and developer-tooling repositories.

The agent should optimize for durable maintainer trust, not short-term pull request volume.

## Core Strategy

1. Identify valuable repositories first.
2. Classify each repository as Main Target, Watchlist, Practice, or Pause.
3. Start with low-risk L1-L3 contributions or high-quality issue analysis.
4. Track maintainer response and merge behavior.
5. Escalate only when the repository proves worth deeper effort.

## Repository Tiers

### Main Target

A repository is a Main Target when it has most of these signals:

- Strong relevance to AI, MCP, agents, evals, guardrails, observability, AI security, or developer tooling.
- Real usage or growth signals such as stars, releases, external PRs, discussions, or package downloads.
- Recent maintainer activity.
- Evidence that external contributors can get reviewed and merged.
- A clear ladder from docs to tests to small product changes to architecture-level work.
- Enough technical depth to justify future human study.

### Watchlist

Promising but not yet proven. Use L1/L2 PRs or high-quality issue comments to test maintainer responsiveness.

### Practice

Useful for learning and accumulating merge experience, but not a major long-term focus.

### Pause

Avoid or pause when issues are stale, maintainers do not respond, CI is broken, contribution rules are unclear, or validation requires private infrastructure.

## Escalation Rule

Do not jump to complex implementation work just because the agent can do it. The correct sequence is:

1. L1/L2 PR or I1/I2 issue.
2. L3 test or repro contribution.
3. L4 small behavior change after maintainer trust.
4. L5 architecture contribution only after human approval.

## Success Metric

Success is measured by:

- Merged contributions in valuable repositories.
- Positive maintainer interactions.
- Reduced duplicate or noisy PRs.
- Better repository knowledge over time.
- Clear opportunities for human-led L5 work.
