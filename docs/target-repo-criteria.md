# Target Repository Criteria

## Main Target Checklist

Use this checklist before investing repeated effort in a repository.

- The repository is relevant to AI, MCP, agents, evals, observability, guardrails, AI security, or developer tooling.
- The project has real users or visible growth.
- Maintainers have been active in the last 30-60 days.
- External contributors receive reviews.
- Issues and pull requests are labeled or triaged.
- Local validation is possible without private infrastructure.
- The project has a contribution ladder from docs to tests to features to architecture.
- Future L5 study would be worth the human's time.

## Watchlist Checklist

A repository can enter Watchlist when it is promising but missing one or more Main Target signals.

Use a small PR, issue comment, or analysis note to test responsiveness.

## Pause Signals

Pause or avoid when:

- Issues are stale with no maintainer response.
- Existing PRs sit unreviewed for long periods.
- The project has unclear contribution rules.
- Maintainers discourage external contributions.
- The required validation depends on private accounts or paid infrastructure.
- The issue is already assigned or clearly claimed.
- The fix already exists on `main` or in an open PR.

## Upgrade Signals

Upgrade a repository when:

- A PR is merged.
- Maintainers respond positively.
- Feedback is concrete and respectful.
- CI passes or failures are actionable.
- A maintainer invites follow-up work.

## L5 Readiness

A repository reaches L5 preparation threshold when:

- It is a Main Target.
- There are at least two accepted contributions or one meaningful L4 merge.
- Maintainers have responded positively.
- There is a clear deep issue or architectural opportunity.
- Human study time has a good expected return.
